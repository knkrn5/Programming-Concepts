If you want to use it inside the Antigravity IDE, you can install it manually using a .vsix file:

For TypeScript 7, Microsoft completely rewrote the engine from scratch in Go. Because Go compiles directly to machine code (unlike JavaScript which requires a virtual machine to interpret it), it executes directly on your OS as a high-performance native binary. This "native" execution is what provides the 10x speed boost.
so the same native machine code binary could have been achieved with c, rust 
In fact, many other modern frontend tools use precisely those languages for the exact same reason (for example, esbuild is written in Go, while SWC and Rolldown are written in Rust).

Go isn't magic here; it just happens to be the language Microsoft picked. C or Rust would have gotten similar (likely even somewhat better) raw performance, since all three compile to native machine code with no VM/interpreter in the loop.
